# WHOOP Data

This folder is auto-generated from the WHOOP Developer API.

- `latest.md`: current snapshot and short rolling trend
- `daily/`: day-level summaries the assistant can reason over quickly
- `raw/`: device-reported historical records stored as JSONL
- `profile.json` and `body_measurement.json`: current profile/body metadata

Treat these files as synced device data, not hand-written notes. If Dill reports something that disagrees with WHOOP, preserve both and label the conflict explicitly.
